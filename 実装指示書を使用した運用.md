# 概要
AIコーディングを利用する際のプロンプトの記法について

# 利用環境
- エディター：VSCode
- 利用AI：Codex（OpenAI）
- VSCodeの拡張機能でCodexをインストールして利用する

# 現状の採用手順
1. 要件定義をmdファイルで作成する
2. 実装手順をmdファイルで作成する
3. 1 と 2 をCodexにレビューさせる
    - 「実装不要」と「実装するにあたり不足している事項のレビュー」を指示に入れておく
4. 指摘事項を潰しきるまで 1 ~ 3 を繰り返す
5. 実装指示
    - 1, 2 で検討した内容をまとめて、以下4点を網羅した実装指示書をCodexに読み込ませる
    1. 改修目的
    2. 要件定義
    3. 具体的な改修箇所と内容の支持
    4. その他の注意事項
        - この項目は必要がなければ省略可

# 注意事項
- 1 ~ 3の段階では、「実装不要」を明記しておかないと、勝手に実装に入ってしまう可能性が高そう
- 5 の実装支持の段階では、資材と実装指示書のみを参照して実装させるようにする

# 過去手順との比較
- 手順を明確化して運用するようにしたら、ビルドエラーや他機能へ影響が出る、などの想定外の事象はかなり減少した
- 実装指示書を作成するまでの手順で、問題点や検討不足の観点があれば洗い出されるので、そちらの価値が高いかもしれない

- アウトプットの質にはまだ課題があり、本手順でもコードの重複や可読性の低さなどは解消されていない
